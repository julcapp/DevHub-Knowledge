# EDP-0013.03 — Authentication Error and Recovery Scenarios

**Код:** UX-0002  
**Версия:** 1.0.0-alpha  
**Статус:** Approved Concept

## Назначение

Документ описывает нестандартные пользовательские сценарии при работе DevHub + Lk_uni.

Цель — обеспечить понятное поведение системы при ошибках авторизации, истечении сессии и восстановлении доступа.

---

# 1. Ошибка входа

Сценарий:

```
Пользователь
 ↓
DevHub
 ↓
Lk_uni
 ↓
Ошибка проверки
```

Экран:

```
Не удалось выполнить вход

Проверьте данные или попробуйте другой способ авторизации.

[Повторить]
[Другой способ входа]
```

Событие записывается в Security Audit.

---

# 2. Истёк срок действия токена

Сценарий:

```
Работа пользователя
 ↓
Access Token expired
 ↓
Проверка Refresh Token
```

Если обновление возможно:

```
Сессия продлена
```

Если невозможно:

```
Требуется повторный вход
```

---

# 3. Истечение сессии по бездействию

Условие:

- 30 минут обычный режим;
- 15 минут защищённый режим.

Экран:

```
Ваша сессия завершена

Причина:
Отсутствие активности

[Войти снова]
```

---

# 4. Восстановление доступа

Сценарий:

```
Пользователь нажимает
"Не могу войти"

↓

Lk_uni Recovery Flow

↓

Подтверждение identity

↓

Новая сессия
```

Поддерживаемые каналы:

- email;
- телефон;
- Telegram;
- другие подключенные identities.

---

# 5. Приглашение участника проекта

Сценарий:

```
Owner проекта
 ↓
Создает приглашение
 ↓
Lk_uni отправляет уведомление
 ↓
Пользователь принимает
 ↓
DevHub создает ProjectMember
```

До принятия:

```
Invitation: Pending
```

После:

```
ProjectMember: Active
```

---

# 6. Изменение роли

Только пользователи с правом управления участниками могут менять роли.

Пример:

```
Александр
изменил роль
Иван

Developer → Architect
```

Событие фиксируется в журнале.

---

# 7. Защищённые действия

Для критичных операций требуется повторное подтверждение:

- удаление проекта;
- передача владения;
- изменение администраторов;
- управление AI ключами;
- экспорт данных.

Сценарий:

```
Пользователь
 ↓
Re-authentication
 ↓
Lk_uni verification
 ↓
Действие разрешено
```
