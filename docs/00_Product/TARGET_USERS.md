# Target Users

**Документ:** TARGET_USERS.md  
**Код:** DOC-0008  
**Версия:** 1.0.0-alpha  
**Статус:** Approved  
**Владелец:** Product Owner  
**Ответственный:** CTO

## 1. Разработчик

Использует DevHub для:

- поиска проектов;
- работы с Git;
- анализа архитектуры;
- подготовки релизов;
- использования AI-помощника;
- создания документации.

## 2. Руководитель проекта

Использует DevHub для:

- просмотра состояния проектов;
- понимания рисков;
- контроля версий;
- анализа активности;
- просмотра инженерной хронологии;
- принятия управленческих решений.

## 3. Архитектор

Использует DevHub для:

- фиксации архитектурных решений;
- анализа структуры проекта;
- сравнения решений;
- оценки технического долга;
- работы с ADR.

## 4. Аналитик

Использует DevHub для:

- требований;
- описания процессов;
- связи требований с реализацией;
- подготовки проектной документации.

## 5. Специалист по качеству

Использует DevHub для:

- контроля соответствия стандартам;
- журналов;
- аудитов;
- рисков;
- прослеживаемости решений;
- подготовки материалов для СМК.

## 6. Product Owner

Использует DevHub для:

- управления продуктовой стратегией;
- приоритизации;
- контроля Roadmap;
- утверждения решений;
- оценки ценности функций.

## 7. Руководитель организации

Использует DevHub для:

- понимания состояния инженерных проектов;
- оценки зрелости команды;
- сохранения корпоративных знаний;
- контроля рисков;
- принятия стратегических решений.

## 8. ИИ-агент

DevHub должен быть понятен не только человеку, но и AI Hub.

ИИ должен уметь читать документы, находить решения, связывать требования с реализацией и готовить рекомендации.
