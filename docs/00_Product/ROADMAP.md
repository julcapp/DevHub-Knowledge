## Ближайший фокус

Главный фокус — вывести DevHub из набора рабочих функций в целостный продукт, где пользователь работает с **проектами**, а не с Git-операциями.

## Milestone M1 — Foundation

**Статус:** In Progress

Цель:

- создать DevHub-Knowledge;
- зафиксировать продуктовые принципы;
- подготовить инженерную базу;
- сохранить утверждённые решения в GitHub.

Поставки:

- FP-001.01 — инициализация DevHub-Knowledge — Done;
- FP-001.02 — Product Foundation — In Progress;
- FP-002 — Engineering Foundation — Planned.

## Milestone M2 — Project Center

**Статус:** Planned

Цель:

- ввести центральную сущность «Проект»;
- создать Паспорт проекта;
- создать Центр действий;
- начать русификацию интерфейса.

Поставки:

- EDP-0010 — Центр проектов;
- EDP-0011 — Русификация интерфейса.

## Milestone M3 — AI Foundation

**Статус:** Planned

Цель:

- создать AI Hub;
- создать интерфейс AI-провайдеров;
- подготовить подключение Qwen, DeepSeek, OpenAI, Ollama и других моделей;
- обеспечить контроль действий ИИ.

Поставки:

- EDP-0012 — AI Hub Foundation;
- EDP-0013 — AI Provider Interface;
- EDP-0014 — AI Chat Panel.

## Milestone M4 — Users and Events

**Статус:** Planned

Цель:

- добавить участников проекта;
- добавить роли и права;
- вести журнал действий;
- подготовить корпоративную эксплуатацию.

Поставки:

- EDP-0015 — Участники проекта;
- EDP-0016 — Журнал событий;
- EDP-0017 — Роли и права.

## Milestone M5 — Engineering Memory

**Статус:** Planned

Цель:

- хранить причины решений;
- вести хронологию проекта;
- связывать требования, ADR, EDP, коммиты и релизы;
- использовать знания в AI Hub.

## Milestone M6 — Enterprise Foundation

**Статус:** Deferred

Цель:

- несколько рабочих пространств;
- корпоративная база знаний;
- ISO/QMS Center;
- командная работа;
- аудит и безопасность.

## Принцип развития

Каждый релиз должен давать пользователю новую практическую ценность и не нарушать долгосрочную архитектуру DevHub.
