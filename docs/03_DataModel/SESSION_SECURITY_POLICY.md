# Session Security Policy

**Код:** SEC-0001  
**Версия:** 1.0.0-alpha  
**Статус:** Approved  
**Связанные документы:** USER_IDENTITY_MODEL.md, LK_UNI_INTEGRATION.md, ROLE_PERMISSION_MATRIX.md

## 1. Назначение

Документ определяет правила управления активными сессиями пользователей DevHub.

Цель:

- защитить данные проектов;
- исключить доступ при оставленном без контроля рабочем месте;
- обеспечить прозрачный аудит входов и выходов.

## 2. Принцип работы

У пользователя есть:

- время жизни сессии;
- время бездействия;
- история активных сессий;
- возможность принудительного завершения.

## 3. Тайм-аут бездействия

Базовое значение для DevHub:

**30 минут без активности**.

После достижения тайм-аута:

1. текущая сессия переводится в состояние `expired`;
2. токен доступа становится недействительным;
3. пользователь перенаправляется на экран повторной авторизации.

## 4. Режимы безопасности

DevHub поддерживает уровни политики:

### Обычный режим

- бездействие: 30 минут;
- срок жизни сессии: 8 часов.

### Повышенный режим

Для проектов с повышенными требованиями:

- бездействие: 15 минут;
- срок жизни сессии: 4 часа.

### Корпоративный режим

Настраивается администратором организации.

Возможные параметры:

- время бездействия;
- обязательная повторная проверка личности;
- разрешённые устройства;
- географические ограничения.

## 5. Экран завершения сессии

Пример:

```
Ваша сессия завершена

Вы не работали в системе более установленного времени.

Для продолжения необходимо выполнить вход.

[Войти снова]
```

## 6. Активные сессии пользователя

Пользователь должен иметь возможность увидеть:

- устройство;
- дату входа;
- последнюю активность;
- IP/регион (при наличии политики);
- статус.

Пример:

```
Chrome Windows
Активна
Последняя активность: 14:32

Телефон Android
Завершена
Вчера 19:10
```

## 7. Аудит событий

Фиксируются:

- успешный вход;
- ошибка входа;
- завершение сессии по тайм-ауту;
- ручной выход;
- отзыв сессии администратором;
- восстановление доступа.

## 8. Интеграция с Lk_uni

Lk_uni отвечает за:

- подтверждение личности;
- управление токенами;
- идентификацию пользователя.

DevHub отвечает за:

- проектные политики сессий;
- требования безопасности конкретного проекта;
- журнал действий внутри проекта.

## 9. Решение

Добавляем управление временем активной сессии как обязательную часть архитектуры авторизации DevHub.

Рекомендованное значение по умолчанию:

**30 минут бездействия с возможностью настройки до 15 минут для защищённых проектов.**
