# EDP-0016.03 — Installer Distribution Policy

**Код:** DEPLOY-0001  
**Версия:** 0.1.0-alpha  
**Статус:** Approved Concept

## Назначение

Документ определяет правила установки DevHub на компьютер пользователя.

Цель — обеспечить понятное размещение продукта, быстрый запуск и отсутствие потери приложения после установки.

---

# 1. Установщик DevHub

При установке пользователь должен получить полноценный продуктовый опыт:

- понятный мастер установки;
- выбор каталога установки;
- создание ярлыков;
- регистрацию компонентов приложения;
- удаление через стандартные средства ОС.

---

# 2. Ярлык на рабочем столе

По умолчанию установщик создаёт:

```
Рабочий стол

└── DevHub
```

Назначение:

- быстрый запуск;
- визуальное присутствие продукта;
- понятный доступ для пользователя.

---

# 3. Меню программ Windows

DevHub должен регистрироваться в меню программ:

```
Пуск

└── DevHub
       ├── DevHub
       ├── Документация
       ├── Проверка обновлений
       └── Удаление DevHub
```

---

# 4. Каталог установки

Стандартный вариант:

```
C:\Program Files\DevHub\
```

Пользовательские данные отдельно:

```
C:\Users\<User>\Documents\DevHub\
```

Принцип:

Программа и данные пользователя не смешиваются.

---

# 5. Иконка приложения

Требования:

- единый фирменный значок;
- используется в ярлыке рабочего стола;
- используется в меню программ;
- используется в окне приложения.

---

# 6. Проверка обновлений

В меню должна быть команда:

```
DevHub

→ Проверить обновления
```

Связь с:

- лицензией;
- версией продукта;
- доступными релизами.

---

# 7. Связь с коммерческой моделью

Установщик должен поддерживать:

- Community Edition;
- Pro License;
- Team License;
- Enterprise License.

После установки выполняется проверка:

```
User
 ↓
License
 ↓
Feature Flags
 ↓
Available Features
```

---

# Решение

Установка DevHub является частью пользовательского опыта и коммерческой модели продукта.

Пользователь должен легко найти, запустить, обновить и удалить продукт.
