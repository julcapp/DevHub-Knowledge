# ADR-0001 — Lk_uni как Identity Provider для DevHub

**Статус:** Approved  
**Дата:** 2026-07-09  
**Владелец решения:** CTO

## Контекст

DevHub развивается как командная инженерная платформа. Для работы нескольких пользователей необходимы:

- регистрация;
- вход;
- восстановление доступа;
- роли;
- права;
- сессии;
- аудит действий.

## Проблема

Реализация собственной системы пользователей внутри DevHub приведёт к дублированию функций и усложнит архитектуру.

## Рассмотренные варианты

### Вариант 1

Создать собственную систему авторизации внутри DevHub.

Минусы:

- дублирование разработки;
- сложность безопасности;
- отдельное сопровождение.

### Вариант 2

Использовать Lk_uni как отдельный сервис идентификации.

Плюсы:

- разделение ответственности;
- повторное использование сервиса;
- единая модель пользователя;
- готовность к корпоративным сценариям.

## Решение

Использовать Lk_uni как внешний Identity Provider для DevHub.

DevHub получает подтверждённого пользователя и управляет бизнес-доступом внутри проектов.

## Последствия

Положительные:

- ускорение разработки;
- единый пользователь во всех продуктах;
- независимое развитие авторизации.

Ограничения:

- требуется стабильный API-контракт;
- необходимо управление версиями интеграции.
