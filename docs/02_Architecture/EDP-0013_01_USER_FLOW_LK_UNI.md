# EDP-0013.01 — DevHub + Lk_uni User Flow

**Статус:** Approved Demo Scenario  
**Версия:** 0.1.0-alpha

## Назначение

Документ описывает пользовательский сценарий работы DevHub с Lk_uni до реализации полноценного коннектора.

Цель — проверить логику взаимодействия человека, сервиса идентификации и платформы проектов.

---

# Сценарий 1. Первый вход

## Шаг 1

Пользователь открывает DevHub.

Экран:

```
DevHub

Управляйте проектами,
знаниями и разработкой

[Войти]
[Создать аккаунт]
```

## Шаг 2

Пользователь выбирает вход.

DevHub передает управление Lk_uni.

## Шаг 3

Lk_uni выполняет:

- проверку пользователя;
- подтверждение identity;
- создание сессии.

## Шаг 4

Lk_uni возвращает DevHub:

- user_id;
- имя;
- статус;
- identities;
- токен сессии.

---

# Сценарий 2. Создание профиля DevHub

После первого входа создаётся локальный профиль:

```
User
 |
 +-- Profile
 |
 +-- Identities
 |
 +-- Sessions
 |
 +-- Project Membership
```

Пользователь получает базовую роль:

```
Новый пользователь
```

---

# Сценарий 3. Создание проекта

Пользователь выбирает:

```
Создать проект
```

DevHub создаёт:

- Project;
- ProjectMember;
- связь User → Project;
- роль Owner.

---

# Сценарий 4. Приглашение участника

Владелец проекта:

```
Добавить участника
```

Выбирает роль:

- Архитектор;
- Разработчик;
- Аналитик;
- Наблюдатель.

После принятия приглашения:

```
User
 ↓
ProjectMember
 ↓
Role
 ↓
Permissions
```

---

# Сценарий 5. Журнал действий

Каждое критичное событие фиксируется:

```
09.07.2026

User created

Login success

Project created

Member invited

Role changed
```

---

# Принцип безопасности

Успешный вход не означает доступ ко всем данным.

Доступ определяется только через:

```
Пользователь
   ↓
Участник проекта
   ↓
Роль
   ↓
Разрешения
```
