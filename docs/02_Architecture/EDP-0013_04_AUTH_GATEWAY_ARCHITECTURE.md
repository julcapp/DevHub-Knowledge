# EDP-0013.04 — Auth Gateway Architecture

**Код:** ARCH-0002  
**Версия:** 0.1.0-alpha  
**Статус:** Approved Concept

## 1. Назначение

Auth Gateway — промежуточный слой между DevHub и Lk_uni.

Он скрывает особенности внешнего сервиса идентификации и предоставляет DevHub единый интерфейс работы с пользователем.

---

# 2. Архитектурная схема

```
Пользователь
    |
    v
DevHub UI
    |
    v
Auth Gateway
    |
    v
Lk_uni
    |
    v
Identity Storage
```

---

# 3. Ответственность компонентов

## Lk_uni

Отвечает за:

- регистрацию;
- подтверждение личности;
- identities;
- управление токенами;
- восстановление доступа;
- безопасность входа.

## Auth Gateway

Отвечает за:

- единый интерфейс авторизации;
- проверку токенов;
- управление жизненным циклом сессии;
- обработку ошибок;
- интеграционные настройки;
- аудит взаимодействия.

## DevHub Core

Отвечает за:

- проекты;
- роли внутри проектов;
- права доступа;
- инженерные действия;
- документы;
- AI Hub.

---

# 5. Модель токенов

Используются:

## Access Token

Назначение:

- короткоживущий доступ к API.

## Refresh Token

Назначение:

- обновление сессии.

## Session ID

Назначение:

- связь пользовательской активности с журналом событий.

---

# 6. Обработка сессий

Auth Gateway контролирует:

- время жизни сессии;
- тайм-аут бездействия;
- отзыв сессии;
- завершение всех сессий.

Политика:

```
30 минут без активности
обычный режим

15 минут
защищённый режим
```

---

# 7. Обработка ошибок

Auth Gateway преобразует технические ошибки в понятные сообщения.

Пример:

Внутри:

```
TOKEN_EXPIRED
```

Пользователю:

```
Время входа истекло.
Выполните повторную авторизацию.
```

# Решение

Auth Gateway является обязательным архитектурным слоем DevHub.

Прямое подключение DevHub Core к Lk_uni не используется.
